# 作品 Concept 總集

> 用途：丟給 Claude Code 改網站文案的內容來源。
> 對應檔案：`src/data/projects.ts`（description / subtitle / overview）、`src/data/sketches.ts`、Hero 與 About 區塊。
> 原則：作品外層只說 concept，並保留技術標籤；角色、成果、連結與「我如何做的」都收進詳情頁。技術亮點（highlights）保留不動，文案是我的第一人稱聲音。

## 總 Concept（Hero / About 用）

**把看不見的，翻譯成感覺得到的。**
I translate the invisible into experience.

我相信前端工程師是翻譯者：在設計與程式之間、在人與機器之間。我做的每一件作品，都是一次翻譯練習——把動作翻譯成音樂、把情緒翻譯成氣味、把設計稿裡的故事翻譯成會動的程式碼。我用程式，將世界表現出來。

## 01 All Things Scored

**Concept 一句話**：動作是看不見的旋律。

**我想表達的**：
世界上所有會動的東西其實都在演奏，只是我們聽不見。一個人走過馬路、一隻貓跳上桌子、樹影在風裡晃——這些動作都有節奏、有高低、有力度。我畫了一條線，任何東西穿過它，就被翻譯成音符。
Computer Vision、Web Audio、樂理，將三個領域，在一條線上演奏。

**網站文案**：
- subtitle：`motion becomes music — 動作是看不見的旋律`
- overview 首句：`世界上所有會動的東西都在演奏，只是我們聽不見——這件作品畫一條線，把穿過它的動作翻譯成旋律。`

## 02 OpenHCI 2026 官方網站

**Concept 一句話**：設計說了一個故障與像素的故事，程式就用動畫把它說完。

**我想表達的**：
好的活動官網不是設計稿的影印機。2026 年的主題是「(AI)dentity」——身份在 AI 時代的故障、解碼與重組。設計團隊用像素和 glitch 講這個故事，我讓程式碼接著講：文字以故障字元的方式解碼進場、loader 用像素風並等字體就位才揭幕。

**網站文案**：
- subtitle：`(AI)dentity — 設計跟程式講同一個故事`
- overview 首句：`設計團隊用故障與像素講「(AI)dentity」的故事，我的工作是讓程式碼把這個故事說完。`

## 03 myBot — AI 體態教練

**Concept 一句話**：把「維持健康」這件難事，翻譯成拍一張照的小事。

**我想表達的**：
體態管理失敗的原因從來不是缺知識，是記錄太麻煩——打開 App、搜尋食物、填表格，第三天就放棄了。我把整件事翻譯成一個最小動作：在 LINE 裡拍張照。照片進來，Gemini Vision 辨識熱量營養素，系統依當日剩餘額度主動建議下一餐。但這個專案真正想說的是工程判斷：AI 產品的關鍵不是到處用 AI，而是知道哪裡**不該**用——明確指令用規則擋、熱量用科學公式算，只把語意理解留給 LLM。溫柔的體驗，底下是冷靜的取捨。

**網站文案**：
- subtitle：`隨傳隨到的 AI 體態教練 — 把難事變成小事`
- overview 首句：`體態管理失敗從來不是缺知識，是記錄太麻煩——所以我把它翻譯成一個最小動作：拍張照。`

## 04 SMILEY 情緒日記

**Concept 一句話**：療癒，從感受情緒開始。感受情緒，從寫日記開始。

**我想表達的**：
這個專題的起點是一個矛盾的觀察：科技讓人隨時連線，卻更孤獨。很多人不是沒有情緒，是說不出口——連對自己都說不出口。SMILEY 讓你只需要寫日記，BERT 模型讀懂主情緒之後，App 用另一種語言回應你：情緒變成顏色、變成一首歌、變成一支精油的氣味。我們不逼使用者「面對」情緒，而是把情緒翻譯成可以感覺的東西，讓察覺自然發生。這是我第一次從痛點、模型、UI Flow 一路做到商業模式，也是我相信「科技應該幫助人靠近自己內心」的起點。

**網站文案**：
- subtitle：`療癒，從感受情緒開始`
- overview 首句：`很多人不是沒有情緒，是說不出口——SMILEY 把日記裡的情緒翻譯成顏色、音樂與氣味，讓察覺自然發生。`

## 05 Click or Check? 媒體識讀遊戲

**Concept 一句話**：媒體倫理很抽象，直到你必須親手按下「分享」。

**我想表達的**：
「要查證再分享」這句話人人會說，但沒有人感覺過它有多難。這個遊戲讓玩家坐上新聞編輯的位子：查證要花時間、聳動標題換流量、信任會流失——每一則新聞都是流量與公信力的拉扯。我們把媒體倫理翻譯成一套你必須親手操作的數值系統，計分表本身就是教學設計：當你為了粉絲數按下那則未查證的新聞，你學到的比十堂課都多。知識用講的會忘，親手做過的選擇不會。

**網站文案**：
- subtitle：`流量 vs 公信力 — 可玩的媒體倫理`
- overview 首句：`「查證再分享」人人會說，但沒人感覺過它有多難——這個遊戲把媒體倫理翻譯成你必須親手權衡的選擇。`

## 06 Playground / Sketches

**Concept 一句話**：翻譯的練習簿——把演算法翻譯成可以玩的東西。

**我想表達的**：
Playground 是我保持手感的地方。沒有客戶、沒有 deadline，只有一個好奇：這段數學、這個隨機性，可以變成什麼感覺？黃金角度變成一座會呼吸的花圃、隨機色票變成一面像素牆。它們不大，但每一件都完整——因為我相信 creative coder 的肌肉是這樣長出來的：持續地把抽象翻譯成體驗，直到變成反射動作。

**各 sketch 文案**：
- 互動花圃：`黃金角度是一串數字，直到它開成一座會跟著滑鼠呼吸的花圃。`
- 鐵皮屋：`鐵皮屋同時帶有人工秩序與時間侵蝕的痕跡。作品以密集排列的圓點模擬浪板表面，再透過不同間距、亮部條紋、缺損率與配色，產生介於建築立面、抽象織物與城市記憶之間的畫面。`

## 07 碩論（進行中）— Meta Ray-Ban 智慧博物館導覽

**Concept 一句話**：讓博物館的知識，跟著你的視線走。

**我想表達的**：
博物館最大的浪費，是知識被鎖在展品旁那塊沒人讀完的說明牌裡。我的碩論用 Meta Ray-Ban Display 眼鏡＋GraphRAG／VideoRAG，把導覽翻譯成一種跟著視線走的對話：你看向哪件作品，知識就在那一刻抵達。研究問題是「資訊出現的時機與形式如何不打斷觀看體驗」——這其實還是同一件事：把看不見的（脈絡、歷史、故事）翻譯成感覺得到的，而且出現得剛剛好。

**網站文案**（若 About 提及碩論）：
- 一句話：`用智慧眼鏡把博物館導覽翻譯成跟著視線走的對話（GraphRAG · VideoRAG · Meta Ray-Ban SDK）。`

## 網站作品資訊層級

- 外層卡片：作品名稱、概念敘述（`description`）、技術標籤、查看詳情。
- 詳情頁：概念標題（`subtitle`）、我如何做的（`overview`）、角色、技術、作品連結、技術亮點。
- 外層不直接放角色、成果列、實作方法、Live Site 或 GitHub 連結。
