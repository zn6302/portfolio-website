## 一、總診斷（先看這段）

現在這個網站是一套**接案 / 個人品牌接案的 Framer 模板**改的，骨架很漂亮（GSAP 卡片翻轉、sticky pin、footer 彈性曲線），但有三個層面的問題：

**一句話：留下漂亮的動畫骨架，把「接案模板」的皮換成「求職作品集」的肉，並且讓文件說真話。**

## 三、結構修改（Structure）詳述

### 目前結構
```
Header → HeroCardJourney(Hero + Skills + About) → Projects(空) → Contact → Footer
```
（另有一堆沒被用到的模板元件：Testimonials、Insights、FAQ、HeroAnimation —— 這些是模板殘留，不是你的東西。）

### 建議結構
```
Header
Hero            ← 定位一句話 + 兩個 CTA（看作品 / GitHub）
Selected Works  ← 主秀！5 支作品，這裡要佔最大篇幅
About + Skills   ← 合併：我是誰 + 真實技能棧
Contact          ← 精簡成 email + GitHub + LinkedIn，拿掉假的聯絡表單
Footer
```

### 具體動作
- **主秀是 Projects，不是 Hero。** 現在 Hero + Skills + About 用掉超長的 scroll 才到作品，太喧賓奪主。把作品往上提，讓人 2 秒內看到你做過什麼。
- **刪掉模板殘留元件**：`Testimonials.tsx`、`Insights.tsx`、`FAQ.tsx`、`Services.tsx`（獨立版）、`HeroAnimation.tsx`。學生求職不需要客戶證言與部落格，留著只會是負擔。
- **Skills 併進 About**，不要獨立一大段。技能是佐證，不是主角。
- **修 anchor 錯亂**：現在 Skills 區塊掛 `id="about"`、真正的 About 沒有 id、導覽列 About 指到 Skills。三者要對齊。

## 八、效能問題（上線即炸的等級）

- **`public/thisME.png` 115MB、`public/thisME.JPG` 84MB**——public/ 內的檔案會原封不動進部署包。已有裁切版 `avatar.jpg`/`services-portrait.jpg` 在用，這兩個原始檔應移出專案。
- **`hero-animation.mp4` 5.6MB**，超過 CLAUDE.md 的 3MB 上限，且 `preload="auto"` + `autoPlay`（[HeroCardJourney.tsx:254-259](src/components/sections/HeroCardJourney.tsx#L254-L259)）意味著**手機也會下載完整影片**——CSS 只是 `display:none` 隱藏卡片旅程，`<video>` 元素照樣抓資源。CLAUDE.md 明定 <768px 不載影片。做法：轉 webm 壓到 3MB 內 + `preload="metadata"` + JS 判斷視窗寬才設 `src`。
- **`about-keyframe.png` 5.1MB**，轉 webp 壓到 300KB 內。
- Google Fonts 用 CSS `@import`（[styles.css:1](src/styles.css#L1)）會串行阻塞首次渲染，改成 index.html 的 `<link rel="preconnect">` + `<link>`。
- JS bundle 335KB（gzip 約 105KB），在 150KB 底線內，這項 OK，不用動。

## 九、RWD 問題

1. **手機看不到第 2–N 個作品**：[Projects.tsx:45-49](src/components/sections/Projects.tsx#L45-L49) 在 ≤1080px 只顯示第一張卡，其餘專案完全不可達。作品集網站的手機版看不到作品，這是必修項——mobile fallback 應改成垂直堆疊卡片列表。

## 十、動畫細節

**先說好的**：[HeroCardJourney.tsx](src/components/sections/HeroCardJourney.tsx) 的工程品質超出學生作品水準——測量式 timeline（不寫死數值）、`invalidateOnRefresh`、`fonts.ready` 後重建、`gsap.context` 完整清理、reduced-motion 支援都有。這段本身就是作品，值得寫進專案介紹，不需要重寫，只需要調長度。

改善點：
1. **卡片旅程太長，喧賓奪主**——從 Hero 滾到 Selected Works 要經過 fullscreen→shrink→Skills 翻轉→About 翻轉，好幾個視窗高度。審查者停留 1–3 分鐘，可能滾到一半就走了。
2. **Projects 空卡片還在跑完整 pin 動畫**（[Projects.tsx:27](src/components/sections/Projects.tsx#L27)）：4 張空白卡要滾 3×0.75 視窗高。`projects.length === 0` 時應直接不 pin、不渲染輪播——這項填完 `projects.ts` 後自然消失，不用獨立處理。
3. **Footer 彈性曲線用 `elastic.out(1.25, 0.34)`**（[Footer.tsx:51](src/components/sections/Footer.tsx#L51)），直接違反 CLAUDE.md「禁止彈跳過度的 spring」。要嘛調成溫和的 `power3.out`，要嘛在 CLAUDE.md 把它列為已知例外。
4. **`html { scroll-behavior: smooth }`（[styles.css:50](src/styles.css#L50)）會干擾 GSAP ScrollToPlugin**：Projects 的上下切換按鈕用 `scrollTo` 動畫時，瀏覽器的 smooth scroll 會疊加造成不精確。建議拿掉，anchor 跳轉交給 GSAP 或 CSS `scroll-behavior` 二選一。
5. **動畫語言數量**：CLAUDE.md 上限四種，但現有的 3D 卡片翻轉旅程、footer 彈性曲線、Projects pin 疊卡都不在那四種清單裡——是又一個要在 §十一 對齊規範時一併處理的點。

**建議**：7/15 前以現況為準、事後改寫 CLAUDE.md（框架級遷移趕不上死線）。色票和字體可以在現有 CSS variables 上低成本替換（`:root` 改 hex + 換 Google Fonts 連結就完成大部分），不需要整套換 Tailwind。對應到 §二的 P5、P7。
